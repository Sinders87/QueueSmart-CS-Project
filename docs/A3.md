## 2. Backend Technologies

We used **Node.js with the built-in HTTP module** to implement the backend for QueueSmart.

Node.js was chosen because it allows us to use JavaScript for both the front end and back end, making development consistent and easier to manage. The built-in HTTP module was used instead of a framework to keep the implementation simple and lightweight. It supports handling API requests, responses, and validations. This setup works well for our QueueSmart design by supporting authentication, queue operations, and data handling without needing a database.
