## 2. Backend Technologies

We used Node.js with the Express.js framework to implement the backend for QueueSmart.

Node.js was chosen because it allows us to use JavaScript for both frontend and backend development, making the system easier to manage and consistent across the project.

Express.js was used to simplify backend development by providing a structured way to handle routes, API requests, and middleware. It allowed us to organize the backend into separate route files (authentication, queue, services, and history), improving scalability and readability.

We used Live Service on VS Code to test each page and authentication.

This setup supports our QueueSmart design by enabling:
- Authentication (login/register)
- Queue management
- Service handling
- History tracking

The modular structure also makes it easier to extend the system in future assignments (e.g., adding a database).
